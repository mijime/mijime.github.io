# madories ツールパネル再設計

## 概要

`packages/madories` のツールパネル（`tool-sheet.tsx`）がボタン密度が高く、視覚的にゴチャついて見える問題を解消する。

## ゴール

- アクションボタンの整理（カテゴリタブ化）
- 主要ツールのタブ化＋サブパネルの縦スクロール対応
- 全体的な視覚的圧迫感の軽減

## 現状の問題

1. アクションボタンが8個横並びで極端に小さい
2. 家具カテゴリー＋アイテムが常時表示で領域を圧迫
3. ボタン密度が高く、余白が少ない
4. 階層がフラットで機能の分類が不明確

## 新設計

### 全体レイアウト

```
┌─ 主要ツールタブ（壁 / 床 / 家具 / 消す / 選択）─┐
├─ サブパネル（縦スクロール領域）──────────────────┤
├─ アクションタブ（編集 / ファイル / 操作）───────┤
└─────────────────────────────────────────────────┘
```

パネル幅は既存の260pxを維持。パネル全体を縦スクロール可能にする。

### 主要ツールタブ

- 壁・床・家具・消す・選択 の5つをタブ化
- アイコン＋ラベル、横並び5等分
- 選択中タブは背景色で強調

### サブパネル

- 壁タブ: 壁種別ボタン（壁/開口部/全窓/半窓/なし）
- 床タブ: 床材2列グリッド（色見本付き）
- 家具タブ: カテゴリー選択 → アイテム一覧（縦スクロール）
- 消す/選択タブ: サブパネルなし

### アクションタブ

- 編集タブ: 戻す・進む（アイコンのみ）
- ファイルタブ: 保存・読込・書出・共有（アイコンのみ）
- 操作タブ: 全体表示・回転・削除（アイコンのみ）

## コンポーネント構造

```
ToolSheet
├─ PrimaryToolTabs
├─ SubPanel
│  ├─ WallSubPanel
│  ├─ FloorSubPanel
│  └─ ItemSubPanel
│     ├─ CategoryTabs
│     └─ ItemList
├─ ActionTabs
│  ├─ EditTab
│  ├─ FileTab
│  └─ OperationTab
└─ ToolPanelContent（既存をリファクタ）
```

## データフロー

- `ToolSheet` は既存の `tool`（`ToolMode`）と `onToolChange` を受け取る
- `PrimaryToolTabs` は `tool.kind` を表示し、クリックで `onToolChange` を呼ぶ
- `SubPanel` は `tool` の内容に応じてサブパネルを切り替え
- `ActionTabs` は独自のタブ状態（`"edit" | "file" | "operation"`）を持つ

## モバイル対応

- ボトムシート形式を維持
- 中身はデスクトップと同じ構造に差し替え

## 変更範囲

- `packages/madories/src/components/tool-sheet.tsx` のみ
- 既存ロジック・型定義・Propsインターフェースは維持

## テスト
